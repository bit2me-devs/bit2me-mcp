What is this?

M

Model Context Protocol

MCP is an open protocol developed by Anthropic that enables AI assistants like Claude to securely connect to external data sources and tools. Think of it as a universal adapter that lets AI access real-world information and perform actions on your behalf.

Local MCP Server Important

This is a local MCP server that runs on your computer. It communicates directly with your Claude Desktop or IDE, executing API calls on your behalf from your local machine.

üîí

Security & Privacy

Your API keys remain on your local machine and are never sent to any third-party server. All communication happens directly between your computer and Bit2Me's API.

Testing with MCP Inspector

üîç

What is MCP Inspector?

MCP Inspector is the official debugging tool for MCP servers. It provides a web interface to test your tools, view responses, and debug issues before using them in Claude or other clients.

  1. 1

    Install MCP Inspector

    Install globally via npm:

    npm install -g @modelcontextprotocol/inspector
  2. 2

    Start the Inspector

    Choose one of these options:

    Recommended Option A: Using NPM Package

    Use the published package (no build required): 

    export BIT2ME_API_KEY=YOUR_BIT2ME_ACCOUNT_API_KEY
export BIT2ME_API_SECRET=YOUR_BIT2ME_ACCOUNT_API_SECRET
npx @modelcontextprotocol/inspector npx -y @bit2me/mcp-server

    Option B: Using Local Repository

    For development or testing unreleased changes: 

    # 1. Clone and build
git clone https://github.com/bit2me-devs/bit2me-mcp.git
cd bit2me-mcp
npm install
npm run build

# 2. Run inspector
export BIT2ME_API_KEY=YOUR_BIT2ME_ACCOUNT_API_KEY
export BIT2ME_API_SECRET=YOUR_BIT2ME_ACCOUNT_API_SECRET
npx @modelcontextprotocol/inspector node build/index.js

    The inspector will automatically open at http://localhost:5173

  3. 3

    Test Your Tools

    The web interface provides:

    • ‚Ä¢ Tools Tab: View all 48 tools, test with parameters, see responses
    • ‚Ä¢ Request/Response Logs: Debug MCP protocol messages and timing
    • ‚Ä¢ Live Testing: Execute tools and see formatted results instantly
  4. 4

Troubleshooting & Errors

üîí Authentication & Security

Error 401: Unauthorized

Your API Keys are invalid or missing permissions.

  • Check for extra spaces in .env
  • Verify keys have "Wallets" and "Trading" permissions

Invalid Signature / Nonce

HMAC signatures require precise timing.

  • Sync your system clock (NTP)
  • Ensure your PC time matches UTC time
üì° Connection & Limits

Error 429: Rate Limit

Too many requests in a short period.

  • get_portfolio_valuation makes 4+ calls at once
  • Wait 60 seconds before retrying

ECONNRESET / Timeout

Network interruption.

  • Check your internet connection
  • Bit2Me API might be under maintenance
‚öôÔ∏è Logic & Usage

"Insufficient Funds"

You are trying to spend more than you have.

  • Check wallet_get_pockets first
  • Remember fees are deducted from the total

"Symbol Not Found"

The trading pair does not exist.

  • Use market_get_ticker to check pairs
  • Format must be BASE/QUOTE (e.g., BTC/EUR)
‚ö†Ô∏è Parameters & Validation

Error 400: Bad Request

Missing or invalid arguments.

  • Verify data types (string vs number)
  • Ensure required fields are present

"Invalid Amount"

Amount format is incorrect.

  • Use dot (.) for decimals, not comma
  • Check min/max limits in market_get_config
üîß Server & Maintenance

Error 500/502

Server-side temporary issue.

  • Bit2Me API might be experiencing load
  • Retry request after a few seconds

Maintenance Mode

System is under scheduled maintenance.

  • Check status.bit2me.com
  • Trading might be paused temporarily
